Soul Onboarding

New Soul installs the iOS app, creates a passkey-backed account, provisions a non-custodial Coinbase Smart Wallet on Base, and establishes their first Konnection via Plaid Link. Passkey is stored in iCloud Keychain; PersonalOS never sees the private key.

    actor Soul
    participant App as iOS App
    participant iCloud as iCloud Keychain
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant CSW as Coinbase Smart Wallet
    participant Base as Base Chain
    participant Plaid as Plaid API

    Soul->>App: Open app, tap "Get Started"
    App->>App: Prompt WebAuthn passkey creation
    App->>iCloud: Store passkey (private key never leaves device)
    iCloud-->>App: Passkey handle confirmed

    App->>BE: POST /auth/register {email, passkey_public_key}
    BE->>DB: INSERT soul_accounts {email, pubkey, status=pending}
    DB-->>BE: soul_id
    BE-->>App: 201 {soul_id, session_token}

    Note over App,CSW: Wallet provisioning (async, may show spinner)
    App->>CSW: initSmartWallet({owner: passkey_public_key, chain: Base})
    CSW->>Base: Deploy ERC-4337 Account contract (counterfactual address)
    Base-->>CSW: wallet_address
    CSW-->>App: {wallet_address, chain_id}

    BE->>DB: UPDATE soul_accounts SET wallet_address, status=active
    DB-->>BE: OK

    BE->>Plaid: POST /link/token/create {user_id, products=[transactions]}
    Plaid-->>BE: {link_token}
    BE-->>App: {link_token}

    App->>Plaid: Open Plaid Link WebView with link_token
    Soul->>App: Selects bank, enters credentials (handled entirely by Plaid)
    Plaid-->>App: onSuccess callback {public_token, institution_id, accounts[]}

    App->>BE: POST /konnections/complete {soul_id, public_token, institution_id, accounts}
    BE->>Plaid: POST /item/public_token/exchange {public_token}
    Plaid-->>BE: {access_token, item_id}
    BE->>DB: INSERT konnections {soul_id, plaid_item_id, access_token_encrypted, status=established, accounts_json}
    DB-->>BE: konnection_id
    BE-->>App: 201 {konnection_id, status=established}
    App-->>Soul: "Your Konnection is live — first Harvest scheduled"
2

Harvest → Scoring → Insight Update

Scheduled Harvest pulls raw Transaktions from Plaid, encrypts them with the Soul's passkey-derived key on-device, writes the encrypted Ledger shard to Arweave via Irys, then triggers on-device Scoring. The device decrypts locally, computes Insight propensity scores, and returns only scores to the backend. Raw data never leaves the device in plaintext, and PersonalOS never holds the decryption key. 

sequenceDiagram
    autonumber
    participant Scheduler as BE Scheduler (cron)
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant Plaid as Plaid API
    participant App as iOS App (on-device)
    participant iCloud as iCloud Keychain
    participant Irys as Irys (Arweave gateway)
    participant Arweave as Arweave

    Note over Scheduler,DB: Triggered by cron — e.g. every 6 hours per Konnection
    Scheduler->>BE: harvestJob({konnection_id})
    BE->>DB: SELECT konnection WHERE id=? AND status=established
    DB-->>BE: {soul_id, plaid_item_id, access_token_encrypted, last_harvested_at}

    BE->>BE: Decrypt plaid access_token (server-side KMS key, NOT Soul's key)
    BE->>Plaid: GET /transactions/get {access_token, start_date=last_harvested_at, end_date=now}
    Plaid-->>BE: {transactions[], total_count, accounts[]}

    BE->>BE: Normalize transactions → Transaktion[] {id, date, merchant, amount, category[]}
    BE->>DB: UPDATE konnections SET last_harvested_at=now, status=syncing

    Note over BE,App: Push raw Transaktion batch to device for encryption
    BE->>App: APNs push: {type=harvest_ready, konnection_id, transaktion_count, batch_id}
    App->>App: Receive background push, wake up
    App->>BE: GET /harvests/{batch_id}/transaktions (auth: session_token)
    BE-->>App: Transaktion[] (plaintext, in-transit TLS only)

    Note over App,iCloud: On-device encryption — key never sent to server
    App->>iCloud: Fetch passkey / derive AES-256 encryption key
    iCloud-->>App: encryption_key (stays in Secure Enclave context)
    App->>App: Encrypt each Transaktion → ciphertext blobs
    App->>App: Build Ledger shard: {soul_id, konnection_id, batch_id, encrypted_transaktions[], timestamp}

    Note over App,Arweave: Write encrypted Ledger shard to Arweave via Irys
    App->>Irys: POST /upload {data: encrypted_ledger_shard, tags: [soul_id, batch_id, version]}
    Irys->>Arweave: Propagate to Arweave network (optimistic — Irys guarantees finality)
    Arweave-->>Irys: tx_id confirmed
    Irys-->>App: {arweave_tx_id, irys_receipt}

    App->>BE: POST /harvests/{batch_id}/ledger-written {arweave_tx_id, irys_receipt, shard_size_bytes}
    BE->>DB: INSERT ledger_shards {soul_id, batch_id, arweave_tx_id, shard_size, written_at}
    BE->>DB: UPDATE konnections SET status=established
    BE->>DB: UPDATE soul_depth {soul_id, increment depth score based on shard_size}
    DB-->>BE: OK
    BE-->>App: 200 {depth_score_updated}

    Note over App,BE: Scoring — all computation on-device
    App->>App: Trigger Scoring job (background task)
    App->>Irys: GET /arweave/{arweave_tx_id} — fetch all Ledger shards for soul
    Irys-->>App: encrypted_ledger_shards[]
    App->>iCloud: Fetch encryption key
    iCloud-->>App: encryption_key
    App->>App: Decrypt Ledger shards in memory (ephemeral — never persisted decrypted)
    App->>App: Run Insight computation models on Transaktion[] locally
    App->>App: Produce Insight scores: [{category, score, confidence, computed_at}]
    App->>App: Wipe decrypted data from memory

    App->>BE: POST /souls/{soul_id}/insights {insights: [{category, score, confidence, computed_at}]}
    BE->>DB: UPSERT soul_insights {soul_id, category, score, confidence, computed_at}
    DB-->>BE: OK
    BE-->>App: 200 {insights_stored: N}

    Note over BE: Scoring-triggered Exchange matching fired here (see Flow 4)
    BE->>BE: emit SCORING_COMPLETE event {soul_id, updated_categories[]}

Exchange Matching: Listing-triggered

A Brand's newly-approved and funded Listing goes live. The Exchange scans all Souls who have a matching Consent for the Listing's Category, whose current Insight score meets the Listing's targeting threshold, and whose Consent Yield floor is at or below the Listing's bid. Matched Souls receive push notifications for each resulting Offer. 

sequenceDiagram
    autonumber
    participant Brand as Brand
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant Base as Base Chain
    participant Exchange as Exchange Engine (BE subsystem)
    participant App as iOS Apps (matched Souls)

    Note over BE,DB: Listing goes live after Admin approval + escrow funded (see Flow 7)
    BE->>DB: UPDATE listings SET status=active WHERE listing_id=?
    DB-->>BE: OK

    BE->>Exchange: emit LISTING_ACTIVATED {listing_id, category, bid_usdc, min_score_threshold, budget_remaining}

    Note over Exchange,DB: Fan-out match across all eligible Souls
    Exchange->>DB: SELECT soul_id, score, wallet_address
FROM soul_insights SI
JOIN consents P ON SI.soul_id = P.soul_id AND SI.category = P.category
JOIN soul_accounts SA ON SI.soul_id = SA.soul_id
WHERE SI.category = listing.category
  AND SI.score >= listing.min_score_threshold
AND P.yield_floor_usdc <= listing.bid_usdc
  AND SA.status = 'active'
  AND NOT EXISTS (SELECT 1 FROM offers WHERE soul_id=SI.soul_id AND listing_id=listing.id AND status != 'expired')
    DB-->>Exchange: matched_souls[] (soul_id, score, wallet_address, yield_floor)

    loop For each matched Soul
        Exchange->>DB: INSERT offers {listing_id, soul_id, bid_usdc, category, content_ref, status=pending, created_at, expires_at}
        DB-->>Exchange: offer_id
        Exchange->>App: APNs push to soul: {type=new_offer, offer_id, brand_name, bid_usdc, category, expires_at}
    end

    Exchange->>DB: INSERT exchange_run_log {listing_id, trigger=listing_activated, matched_count, offers_created, ran_at}
    DB-->>Exchange: OK

    Note over Brand,BE: Brand sees Offer count in dashboard (async)
    Brand->>BE: GET /listings/{listing_id}/stats
    BE->>DB: SELECT offers_count, claims_count, budget_remaining FROM listing_stats
    DB-->>BE: stats
    BE-->>Brand: {offers_sent, claims, budget_remaining_usdc}
4

Exchange Matching: Scoring-triggered

When Scoring completes for a single Soul (see Flow 2), the backend fires a SCORING_COMPLETE event. The Exchange matches that one Soul against all active Listings whose Categories appear in the updated Insight set, checking Consent and Yield floor eligibility. New Offers are delivered via push. 

sequenceDiagram
    autonumber
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant Exchange as Exchange Engine (BE subsystem)
    participant App as iOS App (this Soul)

    Note over BE,Exchange: Triggered at end of Flow 2
    BE->>Exchange: SCORING_COMPLETE {soul_id, updated_categories: [automotive.new_vehicle_purchase, ...]}

    Exchange->>DB: SELECT soul_id, yield_floor_usdc, category
FROM consents
WHERE soul_id = ? AND category IN (updated_categories)
    DB-->>Exchange: soul_consents[] (category, yield_floor_usdc)

    Exchange->>DB: SELECT listing_id, category, bid_usdc, min_score_threshold, budget_remaining, content_ref, brand_name
FROM listings
WHERE status = 'active'
  AND category IN (soul_categories)
  AND budget_remaining > 0
    DB-->>Exchange: candidate_listings[]

    loop For each candidate Listing
        Exchange->>DB: SELECT score FROM soul_insights WHERE soul_id=? AND category=listing.category
        DB-->>Exchange: current_score

        alt score >= listing.min_score_threshold AND consent.yield_floor_usdc <= listing.bid_usdc
            Exchange->>DB: SELECT 1 FROM offers WHERE soul_id=? AND listing_id=? AND status != 'expired'
            DB-->>Exchange: existing_offer?

            alt No existing active Offer
                Exchange->>DB: INSERT offers {listing_id, soul_id, bid_usdc, category, content_ref, status=pending, expires_at}
                DB-->>Exchange: offer_id
                Exchange->>App: APNs push: {type=new_offer, offer_id, brand_name, bid_usdc, category, expires_at}
            else Already has active Offer
                Note over Exchange: Skip — no duplicate Offers per Listing per Soul
            end
        else Score or Yield floor mismatch
            Note over Exchange: Soul not eligible — skip
        end
    end

    Exchange->>DB: INSERT exchange_run_log {trigger=scoring_complete, soul_id, checked_listings, new_offers_created, ran_at}
    DB-->>Exchange: OK
5

Offer → Claim → Yield Settlement

Soul receives a push, opens the Offer in-app, and Claims it. The backend generates a signed Claim authorization which the iOS app submits directly to the Base Chain smart contract. The contract atomically debits Brand escrow, splits USDC to the PersonalOS fee address and Soul's Smart Wallet, and emits a settlement event. PersonalOS backend listens on-chain to confirm and record the Claim. 

sequenceDiagram
    autonumber
    actor Soul
    participant App as iOS App
    participant iCloud as iCloud Keychain
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant Base as Base Chain (Smart Contract)
    participant CSW as Coinbase Smart Wallet

    Soul->>App: Tap push notification
    App->>BE: GET /offers/{offer_id} (auth: session_token)
    BE->>DB: SELECT offer WHERE id=? AND soul_id=? AND status=pending AND expires_at > now()
    DB-->>BE: {offer_id, listing_id, brand_name, bid_usdc, content_ref, category}
    BE-->>App: Offer details + Brand creative content

    App-->>Soul: Display Offer (brand message, bid amount, Category)
    Soul->>App: Tap "Claim — Earn {bid} USDC"

    Note over App,iCloud: Passkey sign — proves Soul identity on-chain
    App->>iCloud: Request passkey signature for Claim authorization
    iCloud-->>App: passkey_signature (WebAuthn assertion)

    App->>BE: POST /offers/{offer_id}/claim {passkey_assertion}
    BE->>BE: Verify passkey assertion against soul's stored pubkey
    BE->>DB: SELECT listing WHERE id=listing_id AND status=active AND budget_remaining >= bid_usdc
    DB-->>BE: listing confirmed
    BE->>DB: UPDATE offers SET status=claiming WHERE id=offer_id
    DB-->>BE: OK

    Note over BE,Base: Backend generates on-chain Claim authorization
    BE->>BE: Sign claim payload: {listing_id, soul_wallet, bid_usdc, fee_usdc, nonce, expiry} with BE signing key
    BE-->>App: {claim_tx_payload, be_signature, escrow_contract_address}

    Note over App,Base: App submits UserOperation via Coinbase Smart Wallet (ERC-4337)
    App->>CSW: prepareUserOp({to: escrow_contract, calldata: claimFunds(payload, be_signature, passkey_sig)})
    CSW-->>App: userOperation
    App->>iCloud: Sign userOperation with passkey
    iCloud-->>App: userOp_signature
    App->>CSW: submitUserOp(userOperation + userOp_signature)
    CSW->>Base: Submit to bundler → include in block

    Note over Base: Atomic on-chain settlement
    Base->>Base: escrow_contract.claimFunds()
  1. Verify BE signature + passkey signature
  2. Check escrow balance >= bid_usdc
  3. Debit listing escrow by bid_usdc
  4. Transfer fee_usdc → PersonalOS fee wallet
  5. Transfer yield_usdc → Soul Smart Wallet
  6. Emit ClaimSettled(listing_id, soul, yield_usdc, fee_usdc, tx_hash)
    Base-->>CSW: tx receipt {tx_hash, block_number, gas_used}
    CSW-->>App: {tx_hash, status=success}

    App->>BE: POST /offers/{offer_id}/claim-confirmed {tx_hash}
    BE->>Base: eth_getTransactionReceipt({tx_hash}) — verify on-chain
    Base-->>BE: receipt {ClaimSettled event confirmed}

    BE->>DB: UPDATE offers SET status=claimed, claimed_at=now(), tx_hash
    BE->>DB: INSERT claims {offer_id, listing_id, soul_id, yield_usdc, fee_usdc, tx_hash, settled_at}
    BE->>DB: UPDATE listings SET budget_remaining = budget_remaining - bid_usdc, claims_count++
    DB-->>BE: OK
    BE-->>App: 200 {yield_usdc, tx_hash, new_wallet_balance_usdc}
    App-->>Soul: "You earned {yield_usdc} USDC! Wallet updated."

    Note over BE: Check if Budget exhausted → triggers Flow 8
    BE->>BE: emit LISTING_CLAIM_SETTLED {listing_id, budget_remaining}

Withdrawal

Soul moves USDC from their non-custodial Coinbase Smart Wallet to fiat via the Coinbase off-ramp. The off-ramp is a Coinbase-hosted flow; PersonalOS mediates only the initial off-ramp session URL and records the withdrawal event after on-chain confirmation. 

sequenceDiagram
    autonumber
    actor Soul
    participant App as iOS App
    participant iCloud as iCloud Keychain
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant CSW as Coinbase Smart Wallet
    participant Base as Base Chain
    participant OffRamp as Coinbase Off-Ramp

    Soul->>App: Tap "Withdraw" in Wallet tab
    App->>BE: GET /souls/{soul_id}/wallet/balance
    BE->>CSW: eth_call balanceOf(soul_wallet_address) on USDC contract
    CSW-->>BE: usdc_balance (on-chain)
    BE-->>App: {usdc_balance, wallet_address}
    App-->>Soul: Display balance, input withdrawal amount

    Soul->>App: Enter amount, confirm
    App->>BE: POST /withdrawals/init {soul_id, amount_usdc}
    BE->>BE: Validate: amount <= usdc_balance, amount >= min_withdrawal
    BE->>DB: INSERT withdrawals {soul_id, amount_usdc, status=initiated, created_at}
    DB-->>BE: withdrawal_id
    BE->>OffRamp: POST /offramp/session {wallet_address, amount_usdc, destination_currency=USD, redirect_url}
    OffRamp-->>BE: {session_url, session_id, expires_at}
    BE->>DB: UPDATE withdrawals SET offramp_session_id=session_id
    BE-->>App: {session_url, withdrawal_id}

    Note over App,OffRamp: Coinbase off-ramp handles KYC, bank linkage, and USDC pull
    App->>OffRamp: Open session_url in SFSafariViewController
    Soul->>OffRamp: Complete Coinbase off-ramp flow (KYC, select bank, confirm)

    Note over OffRamp,Base: Coinbase initiates on-chain USDC pull from Soul wallet
    OffRamp->>Base: Request approval / transfer USDC from soul_wallet_address
    App->>iCloud: Passkey sign transaction approval
    iCloud-->>App: passkey_signature
    App->>CSW: Sign and submit USDC transfer UserOperation
    CSW->>Base: Transfer USDC: soul_wallet → Coinbase custody address
    Base-->>CSW: tx_hash

    Note over OffRamp,Soul: Coinbase initiates ACH / wire bank transfer (T+1 to T+3)
    OffRamp->>OffRamp: Initiate ACH transfer to Soul's linked bank

    OffRamp->>BE: Webhook POST /webhooks/offramp {session_id, status=usdc_received, tx_hash}
    BE->>DB: UPDATE withdrawals SET status=processing, onchain_tx_hash=tx_hash
    DB-->>BE: OK

    OffRamp->>BE: Webhook POST /webhooks/offramp {session_id, status=fiat_sent, bank_reference}
    BE->>DB: UPDATE withdrawals SET status=completed, fiat_sent_at=now(), bank_reference
    DB-->>BE: OK
    BE->>App: Push notification: "Withdrawal of ${amount} complete — funds arriving in 1-3 days"
    App-->>Soul: Withdrawal confirmed

Brand Onboarding

A Brand signs up, subscribes to the platform, creates a Listing targeting a Signal Type, submits it for Admin review, and once approved funds the USDC escrow on Base. Only after escrow is confirmed on-chain does the Listing go live and trigger Exchange matching (Flow 3). 

sequenceDiagram
    autonumber
    actor Brand
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant Admin as Platform Admin
    participant Base as Base Chain
    participant Exchange as Exchange Engine

    Brand->>BE: POST /auth/brand/register {company_name, email, password}
    BE->>DB: INSERT brand_accounts {company_name, email, password_hash, status=pending_subscription}
    DB-->>BE: brand_id
    BE-->>Brand: 201 {brand_id, session_token}

    Note over Brand,BE: Subscription — platform access fee (fiat, via Stripe or similar)
    Brand->>BE: POST /brands/{brand_id}/subscribe {plan, payment_method_token}
    BE->>BE: Process subscription payment
    BE->>DB: UPDATE brand_accounts SET status=active, subscription_plan, subscribed_at
    DB-->>BE: OK
    BE-->>Brand: 200 {subscription_confirmed}

    Note over Brand,DB: Brand creates a Listing
    Brand->>BE: POST /listings {category, bid_usdc, min_score_threshold, content: {headline, body, cta_url}, budget_usdc, target_audience_notes}
    BE->>BE: Validate: category in allowed set, bid_usdc >= platform_minimum, budget_usdc >= bid_usdc * 10
    BE->>DB: INSERT listings {brand_id, category, bid_usdc, min_score_threshold, content_json, budget_usdc, budget_remaining=budget_usdc, status=pending_review, created_at}
    DB-->>BE: listing_id
    BE-->>Brand: 201 {listing_id, status=pending_review}

    Note over BE,Admin: Admin review queue
    BE->>Admin: Notify: new Listing pending review {listing_id, brand_name, category, content_summary}
    Admin->>BE: GET /admin/listings/{listing_id}
    BE->>DB: SELECT listing + brand details
    DB-->>BE: listing_full
    BE-->>Admin: Listing for review

    alt Admin approves
        Admin->>BE: POST /admin/listings/{listing_id}/approve {reviewer_notes}
        BE->>DB: UPDATE listings SET status=approved_pending_funding, approved_at, reviewer_id
        DB-->>BE: OK
        BE-->>Brand: Email + webhook: "Listing approved — please fund escrow to go live"

        Note over Brand,Base: Brand funds escrow on Base Chain
        Brand->>BE: POST /listings/{listing_id}/fund-escrow
        BE->>Base: Call escrow_contract.depositBudget(listing_id, budget_usdc) — expect Brand to send tx
        BE-->>Brand: {escrow_contract_address, listing_id, amount_usdc, deposit_calldata}

        Brand->>Base: Submit USDC approval + depositBudget transaction from Brand wallet
        Base->>Base: Transfer budget_usdc USDC from Brand → escrow contract
Lock funds under listing_id
Emit BudgetDeposited(listing_id, brand, amount_usdc)

        Base->>BE: On-chain event listener: BudgetDeposited {listing_id, amount_usdc, tx_hash}
        BE->>DB: UPDATE listings SET status=active, escrow_tx_hash=tx_hash, activated_at=now()
        DB-->>BE: OK
        BE-->>Brand: "Listing is LIVE — Exchange matching begins now"

        BE->>Exchange: emit LISTING_ACTIVATED {listing_id} → triggers Flow 3

    else Admin rejects
        Admin->>BE: POST /admin/listings/{listing_id}/reject {reason}
        BE->>DB: UPDATE listings SET status=rejected, rejection_reason, reviewed_at
        DB-->>BE: OK
        BE-->>Brand: Email: "Listing rejected — reason: {reason}. You may revise and resubmit."
    end
8

Brand Listing Exhaustion

Each Claim deducts the bid from the Listing's on-chain escrow balance. When the remaining Budget falls below the bid amount, the smart contract can no longer pay future Claims. The contract emits a BudgetExhausted event, the Exchange Engine deactivates the Listing, pending Offers are expired, and the Brand is notified. Any dust (remaining USDC too small to cover a full Claim) is returned to the Brand. 

sequenceDiagram
    autonumber
    participant Base as Base Chain (Smart Contract)
    participant BE as PersonalOS Backend
    participant DB as PersonalOS DB
    participant Exchange as Exchange Engine
    participant App as iOS Apps (pending-offer Souls)
    actor Brand

    Note over Base,BE: Each successful Claim (Flow 5) reduces escrow. Final Claim may exhaust budget.
    Base->>Base: claimFunds() executes
budget_remaining = budget_remaining - bid_usdc
IF budget_remaining < bid_usdc:
  emit BudgetExhausted(listing_id, dust_usdc)
  Set listing status = exhausted in contract

    Base->>BE: On-chain event: BudgetExhausted {listing_id, dust_usdc, block_number, tx_hash}

    BE->>DB: UPDATE listings SET status=exhausted, exhausted_at=now(), final_budget_remaining=dust_usdc
    DB-->>BE: OK

    Note over BE,Exchange: Stop the Exchange from creating new Offers for this Listing
    BE->>Exchange: emit LISTING_EXHAUSTED {listing_id}
    Exchange->>Exchange: Remove listing_id from active Listings index

    Note over BE,DB: Expire all pending Offers for this Listing
    BE->>DB: UPDATE offers SET status=expired, expired_at=now()
WHERE listing_id=? AND status=pending
    DB-->>BE: expired_offers_count

    Note over App: Notify Souls whose pending Offer just expired
    BE->>DB: SELECT soul_id FROM offers WHERE listing_id=? AND status=expired AND expired_at=now()
    DB-->>BE: affected_souls[]
    loop For each affected Soul
        BE->>App: APNs push: {type=offer_expired, offer_id, reason="Brand budget exhausted"}
    end

    Note over Base,Brand: Return dust USDC to Brand wallet
    alt dust_usdc > 0
        Base->>Base: Transfer dust_usdc → brand_wallet_address
Emit DustReturned(listing_id, brand_wallet, dust_usdc)
        Base->>BE: On-chain event: DustReturned {listing_id, amount_usdc, tx_hash}
        BE->>DB: INSERT brand_refunds {brand_id, listing_id, amount_usdc, tx_hash, refunded_at}
        DB-->>BE: OK
    end

    BE->>Brand: Email notification:
"Your Listing [{listing_id}] has exhausted its budget.
{total_claims} Claims settled. {dust_usdc} USDC returned.
Create a new Listing or top up to continue reaching Souls."

    Note over Brand,BE: Brand may choose to create a new Listing or top up
    Brand->>BE: POST /listings/{listing_id}/top-up {additional_budget_usdc}
    BE->>DB: SELECT listing WHERE id=? AND brand_id=? AND status=exhausted
    DB-->>BE: listing
    BE-->>Brand: {escrow_contract_address, top_up_calldata}
    Brand->>Base: Submit USDC depositBudget transaction
    Base->>Base: Add USDC to escrow
Reset listing status = active
Emit BudgetDeposited(listing_id, additional_usdc)
    Base->>BE: On-chain event: BudgetDeposited {listing_id, amount_usdc}
    BE->>DB: UPDATE listings SET status=active, budget_remaining=budget_remaining+additional_usdc
    DB-->>BE: OK
    BE->>Exchange: emit LISTING_ACTIVATED {listing_id} → resumes Exchange matching (Flow 3)
    BE-->>Brand: "Listing reactivated — Exchange matching resumed"
